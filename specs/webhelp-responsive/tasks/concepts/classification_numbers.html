  • Entity modeling

    The transformation from MARC to Linked Art requires that both records and access points within records be modeled and entified in order to create a normalized representation of the data.

    • LUX top-level entities

      Eleven top-level entities are required for use in LUX.

      • Record-level entities

        Entities that correspond to resources held by the library (books, journals, databases, archival collections, etc.).

      • Related entities

        Entities that represent the people, places, concepts, etc., that are connected to record-level resources.

        • Creators, contributors, standalone works, simple subject/genre headings, and associated places

          Related resources that represent a single entity type.

        • Complex subject/genre headings

  - preserve:
    - period

  1. For each unique 050a or 090a value, generate and store a top-level concept resource, identified by an IRI.
  2. Create a key for matching.
    1. Replace any punctuation characters that are not a period (.) with a whitespace character.
    2. Normalize and match/merge each unique string value.
    3. After normalizing, use an appropriate regular expression to test the value of 050a or 090a to see whether it is an LC classification number.
      [a-z]+[0-9]+([.][a-z]+[0-9]+)?
    4. For values that match the test, add a classified_as for Library of Congress classification, as in the example below.
    5. For now, skip any values that do not match the regular expression.
